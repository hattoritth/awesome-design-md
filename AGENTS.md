# AGENTS.md

## Scope

This public repository collects `DESIGN.md` files and related notes for reusable
website and product design-system references.

## Operating Rules

- Keep guidance generic, public-safe, and reusable.
- Do not add private project details, account data, credentials, customer data, or
  unpublished internal design materials.
- Preserve source attribution and avoid copying more source material than needed.
- Keep one design-system reference focused on one product, site, or visual system.
- Mark uncertain visual interpretations, token names, or implementation details as
  `TBD` or `needs verification` rather than guessing.

## Skill Entry Points

- Use `.agents/skills/design-reference-maintenance/SKILL.md` for adding,
  reviewing, or normalizing `DESIGN.md` references and related docs.

## Verification

- Reread changed Markdown files.
- Check links and source names.
- Confirm no private screenshots, credentials, raw proprietary assets, or unrelated
  files are included.
- Keep root instructions short; put long procedures or checklists in docs or skills.
